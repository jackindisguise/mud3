# Commit Guide
Follow these rules when generating commit messages.

## Do
1. Describe what you DID.
2. Use conventional-commit syntax.
3. Message should start with a lowercase letter.
4. In the body, list all of the changes.
    1. Begin each entry in the list with a conventional-commit prefix.
    2. Use a single blank line after the title.
    3. Keep all bullets in a single compact block.
5. Use past tense when describing changes. "Added," not "add."
6. Be concise.
7. Write the commit message to a file so you can use -F instead of -m

## Do NOT
1. Describe what you were TOLD to do.
2. Use emojis.
3. Add extra line breaks between bullet points.
4. Start each entry in the commit body with a hyphen.
